## วิธีอัปโหลดขึ้น GitHub

### ขั้นตอนที่ 1: สร้างบัญชี GitHub
ไปที่ https://github.com และสมัครบัญชีฟรี

### ขั้นตอนที่ 2: สร้าง Repository ใหม่
1. คลิก **"New repository"** (ปุ่มสีเขียว)
2. ตั้งชื่อ repository เช่น `electron-orbit-quest`
3. ติ๊ก **"Public"**
4. คลิก **"Create repository"**

### ขั้นตอนที่ 3: อัปโหลดไฟล์
1. ในหน้า repository ใหม่ คลิก **"uploading an existing file"**
2. ลากไฟล์ทั้ง 4 ไฟล์เข้าไปในกล่อง
3. คลิก **"Commit changes"**

## วิธีเปิด GitHub Pages

1. ใน repository ของคุณ คลิก **Settings** (แถบด้านบน)
2. เลื่อนลงไปที่ **"Pages"** ในเมนูซ้าย
3. ใต้ **"Source"** เลือก:
   - Branch: **main**
   - Folder: **/ (root)**
4. คลิก **Save**
5. รอสักครู่ จากนั้นลิงก์เว็บไซต์จะปรากฏ เช่น:
   ```
   https://username.github.io/electron-orbit-quest/
   ```
6. แชร์ลิงก์นี้ให้นักเรียนได้เลย!

## วิธีเพิ่มข้อสอบ

เปิดไฟล์ `script.js` ค้นหา `const questionBank = [` แล้วเพิ่มข้อสอบใหม่ในรูปแบบนี้:

### ข้อสอบแบบ MCQ (Multiple Choice)
```javascript
{
  id: 'q99',           // ID ไม่ซ้ำกัน
  level: 1,            // ด่านที่เกี่ยวข้อง (1-10)
  topic: 'shellConfiguration',  // หัวข้อ
  type: 'mcq',         // ประเภทข้อสอบ
  question: 'Na มีเลขอะตอมเท่าไร?',
  options: ['9','10','11','12'],
  answer: '11',
  explanation: 'Na มีเลขอะตอม 11 ดังนั้นจึงมีโปรตอน 11 ตัว',
  hint: 'เลขอะตอม = จำนวนโปรตอน',
  visualType: 'atomModel',
  visualData: 'Na'
},
```

### ข้อสอบแบบ Ordering (เรียงลำดับ)
```javascript
{
  id: 'q100',
  level: 9,
  topic: 'atomicRadiusTrend',
  type: 'ordering',
  question: 'เรียงขนาดจากใหญ่ไปเล็ก: Li, Na, K',
  pool: ['Li','Na','K'],
  answer: ['K','Na','Li'],
  explanation: 'ในหมู่เดียวกัน ลงมาขนาดใหญ่ขึ้น: K > Na > Li',
  hint: 'ในหมู่เดียวกัน ลงมาล่างขนาดใหญ่ขึ้น',
  visualType: 'atomicRadiusChart',
  visualData: ['K','Na','Li']
},
```

จากนั้นเพิ่ม ID ข้อสอบนั้นในด่านที่ต้องการ ใน `LEVELS_CONFIG`:
```javascript
{ id:1, name:'Atomic ID', questionIds:['q01','q02',...,'q99'], maxScore:XX },
```

## วิธีเพิ่มธาตุ

เปิด `script.js` ค้นหา `const ELEMENTS = {` แล้วเพิ่มธาตุใหม่ เช่น Sc (สแกนเดียม):

```javascript
Sc: {
  z: 21, sym: 'Sc', nameTH: 'สแกนเดียม',
  e: 21, 
  shell: [2,8,9,2],       // การจัดเรียงแบบเชลล์
  orbital: '1s² 2s² 2p⁶ 3s² 3p⁶ 3d¹ 4s²',
  valence: 2,
  period: 4, group: 3,
  commonIon: 'Sc³⁺', ionCharge: 3,
  ionE: 18, ionShell: [2,8,8],
  atomicR: 184, ionR: 74.5,
},
```

**ฟิลด์สำคัญ:**
| ฟิลด์ | ความหมาย |
|-------|----------|
| `z` | เลขอะตอม |
| `sym` | สัญลักษณ์ |
| `nameTH` | ชื่อภาษาไทย |
| `e` | จำนวนอิเล็กตรอน |
| `shell` | การจัดเรียงแบบเชลล์ [K,L,M,N] |
| `orbital` | การจัดเรียงแบบหลัก |
| `valence` | เวเลนซ์อิเล็กตรอน |
| `commonIon` | ไอออนที่พบบ่อย |
| `ionCharge` | ประจุของไอออน (+/-) |
| `ionE` | จำนวน e⁻ ของไอออน |
| `ionShell` | การจัดเรียงเชลล์ของไอออน |
| `atomicR` | รัศมีอะตอม (pm) |
| `ionR` | รัศมีไอออน (pm) |

## วิธีดาวน์โหลดคะแนน

### สำหรับนักเรียน
1. เล่นจนจบ → ดูใบประกาศ
2. กด **"⬇️ ดาวน์โหลด CSV"**
3. ไฟล์ CSV จะถูกดาวน์โหลดไปยังเครื่อง

### สำหรับครู (Admin)
1. กดปุ่ม **"Admin"** เล็กๆ ที่ด้านล่างหน้าแรก
2. ใส่รหัสผ่าน: `teacher2569`
3. ในหน้า Admin กด **"⬇️ CSV"** เพื่อดาวน์โหลดข้อมูลนักเรียนทั้งหมด

### หมายเหตุสำคัญ
ข้อมูลคะแนนจะเก็บอยู่ใน **localStorage ของเครื่องที่เล่น** เท่านั้น  
ถ้านักเรียนเล่นบนมือถือของตัวเอง ครูจะเห็นคะแนนเฉพาะเครื่องที่ครูนั่งอยู่เท่านั้น  
แนะนำให้นักเรียน **แคปจอใบประกาศส่งครู**

## วิธีแก้ไขข้อมูลธาตุและไอออน

ค้นหา `const ELEMENTS = {` ใน `script.js`

แต่ละธาตุมีฟิลด์ไอออนดังนี้:
```javascript
Na: {
  // ... ข้อมูลธาตุปกติ ...
  commonIon: 'Na⁺',    // สัญลักษณ์ไอออน
  ionCharge: 1,         // ประจุ (+บวก / -ลบ)
  ionE: 10,             // จำนวน e⁻ หลังเป็นไอออน
  ionShell: [2,8],      // การจัดเรียงเชลล์ของไอออน
  atomicR: 186,         // รัศมีอะตอม (pm)
  ionR: 102,            // รัศมีไอออน (pm)
},
```

**ตัวอย่างเปลี่ยนไอออนของ Ca:**
```javascript
Ca: {
  // ...
  commonIon: 'Ca²⁺',
  ionCharge: 2,
  ionE: 18,
  ionShell: [2,8,8],
  atomicR: 197,
  ionR: 100,
},
```

## วิธีเพิ่มภาพจำลองในโจทย์

ในข้อสอบแต่ละข้อ มีฟิลด์ `visualType` และ `visualData`

### ประเภทภาพที่รองรับ

| visualType | ความหมาย | visualData |
|-----------|----------|-----------|
| `atomModel` | แบบจำลองอะตอม | สัญลักษณ์ธาตุ เช่น `'Na'` |
| `shellModel` | แบบจำลองเชลล์ | สัญลักษณ์ธาตุ เช่น `'Cl'` |
| `ionModel` | แบบจำลองไอออน | สัญลักษณ์ธาตุ เช่น `'Na'` |
| `atomicRadiusChart` | กราฟขนาดอะตอม | อาร์เรย์ธาตุ เช่น `['Na','Mg','Cl']` |
| `ionicRadiusCompare` | เปรียบขนาดอะตอม-ไอออน | อาร์เรย์ เช่น `['Na','Na+']` |
| `isoelectronicChart` | ไอโซอิเล็กโทรนิก | อาร์เรย์ไอออน |
| `aufbauDiagram` | แผนภาพ Aufbau | ไม่ต้องใส่ |

**ตัวอย่าง:**
```javascript
{
  id: 'q_custom',
  visualType: 'atomicRadiusChart',
  visualData: ['Na','Mg','Al','Si','P','S','Cl'],
  // ...
}
```

## วิธีปรับคลังข้อสอบ Final Boss

Final Boss (ด่าน 7) จะ**สุ่มข้อสอบ 30 ข้อ**จาก `questionBank` ทั้งหมด

เพื่อควบคุมว่าข้อสอบไหนจะออกใน Final Boss:
1. เพิ่ม `level: 7` ในข้อสอบที่ต้องการให้ออก
2. หรือไม่ใส่ `level` เลย ก็จะถูกสุ่มทั้งหมด

หากต้องการปรับจำนวนข้อ (เช่น เปลี่ยนเป็น 20 ข้อ):
```javascript
// ใน startLevelGame() บริเวณ
state.currentQuestions = shuffle(pool).slice(0, 30);
// เปลี่ยน 30 เป็น 20
state.currentQuestions = shuffle(pool).slice(0, 20);
```

## ข้อจำกัดของ localStorage

| ข้อจำกัด | รายละเอียด |
|----------|-----------|
| **เก็บเฉพาะเครื่อง** | ข้อมูลอยู่ในเบราว์เซอร์เครื่องนั้นเท่านั้น |
| **ขนาดจำกัด** | ~5MB ต่อ domain (เพียงพอสำหรับการใช้งาน) |
| **ล้างได้** | ถ้านักเรียนล้าง cache เบราว์เซอร์ ข้อมูลหาย |
| **ไม่ข้ามอุปกรณ์** | มือถือกับคอมพิวเตอร์เก็บข้อมูลแยกกัน |

**คำแนะนำสำหรับครู:**
- ให้นักเรียน**แคปหน้าจอใบประกาศ**ส่งครูเป็นหลัก
- ถ้าต้องการเก็บคะแนนส่วนกลาง ต้องใช้ Google Sheets + Google Apps Script แทน
